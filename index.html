Introducción

Objetivo

Este informe tiene el fin de hacer un análisis profundo del rendimiento de algunas variantes de algoritmos de ordenamiento. Para lograr esto, fue necesario obtener datos acerca de las diferentes variables que tiene cada agoritmo.

Generación de los datos

Los datos con los que se desarrolló este informe fueron obtenidos mediante sistemas de “benchmarking” programados (Java) por el autor, cuidando mucho la eficiencia de los mismos. Estos sistemas toman el algoritmo especificado y realizan pruebas con diversos tamaños de arreglos, tipo de datos y otras variables que sean específicas del algorítmo.

Sets de parámetros

Estos parámetros incluyen las siguientes verticales: dataType (tipo de dato del set a ordenar, puede ser String o Integer), dataOrder (orden del set de datos antes de ser ordenado, Ascendiente, Descendiente, Random) y un N (tamaño del set de datos). Algunos algorítmos específicos requieren de parametros diferentes como secuencias en el caso del ShellSort o un M en el caso del QuickCut, que se explicarán posteriormente. Para cada sorter se realizaron pruebas de diversas combinaciones de las variables anteriormente mencionadas. De esas pruebas se obtuvo información de las siguientes variables a analizar: Comparisons (cantidad de comparaciones que realizo el sorter), Swaps (cantidad de intercambios de elementos que realizo el sorter) y Time(ms) (tiempo que tardo el sorter en ordenar el set de datos, expresado en milisegundos). Para el caso del ShellSort también se diferenció según la secuencia evaluada y el QuickCut se diferenció por M.

Bubble Sorter

La ordenación de burbuja (Bubble Sort en inglés) es un sencillo algoritmo de ordenamiento. Funciona revisando cada elemento de la lista que va a ser ordenada con el siguiente, intercambiándolos de posición si están en el orden equivocado.

Rendimiento teórico

  • Casos promedio y peor:
    • Comparaciones: ~N2/2
    • Intercambios: ~N2/2
  • Mejor caso:
    • Comparaciones: ~N2/2
    • Intercambios: 0

Insertion Sorter

El ordenamiento por inserción (insertion sort en inglés) es una manera muy natural de ordenar para un ser humano. Inicialmente se tiene un solo elemento, que obviamente es un conjunto ordenado. Después, cuando hay k elementos ordenados de menor a mayor, se toma el elemento k+1 y se compara con todos los elementos ya ordenados, deteniéndose cuando se encuentra un elemento menor (todos los elementos mayores han sido desplazados una posición a la derecha) o cuando ya no se encuentran elementos (todos los elementos fueron desplazados y este es el más pequeño). En este punto se inserta el elemento k+1 debiendo desplazarse los demás elementos.

Rendimiento teórico

  • Peor caso:
    • Comparaciones: ~N2/2
    • Intercambios: ~N2/2
  • Caso promedio:
    • Comparaciones: ~N2/4
    • Intercambios: N2/4
  • Mejor caso:
    • Comparaciones: N-1
    • Intercambios: 0

Selection Sorter

El ordenamiento por selección (Selection Sort en inglés) es un algoritmo de ordenamiento que busca el mínimo elemento de la lista, lo intercambia por el primero, busca el siguiente mínimo, lo intercambia por el segundo y así en general. Buscar el mínimo elemento entre a posición i y el final de la lista, intercambiar el mínimo con el elemento de la pocisión i.

Rendimeinto teórico

  • Comparaciones: ~N2/2

Comparación Bubble, Selection e Inseriton (dataType: int, dataOrder: random)

En el caso del Tiempo, el Selection es el que mejor performa, siendo el que menor tiempo de ejecución tiene en Ns grandes. Para las Comparaciones, los tres tipos de sorters performan de una forma similar. La diferencia drástica se da en los Intercambios, donde el Selection sorter es el que mejor performa por lejos. Es por ello que de estos tres sorters, es válido concluir que el mejor de ellos es el Selection.

Shell Sorter

El Shell sort es una generalización del ordenamiento por inserción, teniendo en cuenta dos observaciones: El ordenamiento por inserción es eficiente si la entrada está “casi ordenada”. El ordenamiento por inserción es ineficiente, en general, porque mueve los valores sólo una posición cada vez. El algoritmo mejora el ordenamiento por inserción comparando elementos separados por un espacio de varias posiciones. Esto permite que un elemento haga “pasos más grandes” hacia su posición esperada. Los pasos múltiples sobre los datos se hacen con tamaños de espacio cada vez más pequeños. El último paso del Shell sort es un simple ordenamiento por inserción, pero para entonces, ya está garantizado que los datos del vector están casi ordenados.

Quick Sorter

El ordenamiento rápido trabaja de la siguiente forma: Elige un elemento del arreglo de elementos a ordenar, al que llamaremos pivote. Resitua los demás elementos de la lista a cada lado del pivote, de manera que a un lado queden todos los menores que él, y al otro los mayores. Los elementos iguales al pivote pueden ser colocados tanto a su derecha como a su izquierda, dependiendo de la implementación deseada. En este momento, el pivote ocupa exactamente el lugar que le corresponderá en la lista ordenada. La lista queda separada en dos sublistas, una formada por los elementos a la izquierda del pivote, y otra por los elementos a su derecha. Repite este proceso de forma recursiva para cada sublista mientras éstas contengan más de un elemento. Una vez terminado este proceso todos los elementos estarán ordenados.

Merge Sorter

El algoritmo de ordenamiento por mezcla funciona de la siguiente manera: Si la longitud de la lista es 0 ó 1, entonces ya está ordenada. En otro caso: Divide la lista desordenada en dos sublistas de aproximadamente la mitad del tamaño. Ordena cada sublista recursivamente aplicando el ordenamiento por mezcla. Por último, une las dos sublistas en una sola lista ordenada.
